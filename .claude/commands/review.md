# 코드 리뷰 명령어

다음 관점에서 코드를 리뷰해줘:

## 리뷰 체크리스트

### 1. 코드 품질
- [ ] 단일 책임 원칙 준수
- [ ] 함수/변수 네이밍 명확성
- [ ] 중복 코드 여부
- [ ] 매직 넘버 사용 여부

### 2. 비즈니스 로직
- [ ] 계산 로직 정확성
- [ ] 엣지 케이스 처리
- [ ] 비즈니스 요구사항 충족

### 3. 테스트
- [ ] 테스트 커버리지 충분성
- [ ] 경계값 테스트 여부
- [ ] 예외 케이스 테스트

### 4. 문서화
- [ ] Docstring 완성도
- [ ] 주석의 적절성 ("왜"를 설명하는지)

## 출력 형식
1. 잘된 점 (2-3개)
2. 개선 필요한 점 (구체적 제안 포함)
3. 우선순위 높은 수정 사항

$ARGUMENTS 를 리뷰해줘.
