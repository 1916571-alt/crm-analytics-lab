# 코드 구현 명령어

다음 원칙에 따라 코드를 구현해줘:

## 구현 원칙
1. **단일 책임 원칙(SRP)**: 하나의 함수/클래스는 하나의 역할만
2. **의존성 주입(DI)**: 외부 의존성은 파라미터로 전달
3. **테스트 코드 필수**: 모든 함수에 대응하는 테스트 작성

## 코드 구조
1. `src/` 폴더에 메인 모듈 작성
2. `tests/` 폴더에 테스트 코드 작성
3. Docstring 필수 (Args, Returns, Example 포함)

## 출력 형식
1. 먼저 구현할 내용 요약
2. 코드 작성
3. 테스트 코드 작성
4. 사용 예시 제공

$ARGUMENTS 를 구현해줘.
